## 目标范围
- 对 `pages` 与 `pages-zh` 两目录的所有章节，从第5章起逐章核对英文与中文内容（包含 `.adoc` 与 `.xml`）。
- 重点确保：结构、术语、示例命令与输出、Mermaid 图、表格、TIP/NOTE、交叉引用锚点完全一致。

## 方法与规则
- 文本一致性：逐节对照英文原文，修正中文标题、小节名、段落与列表；保持语义一致而非直译。
- 结构一致性：保证章节层级、锚点 `[[...]]`、小节顺序、表格列数/列名完全一致。
- 符号与转义：统一并校正易错符号渲染，如 `+%`、`+|`、`[*]T`、`[:0]T`、`pass:[...]`、内联反引号与转义，确保在 AsciiDoc 与 DocBook XML 中渲染正确。
- 示例一致性：`Run/Output` 标签与示例命令、示例输出在中英文保持一一对应；术语本地化但数值与代码保持原样。
- Mermaid 一致性：确认图代码在中英文内容一致，仅保留必要的中文注释或标题，不改变图逻辑。
- 不新增文件：仅编辑现有 `.adoc`/`.xml`；不改动示例源文件与路径。

## 具体步骤
1. 章节映射与发现：枚举 `pages/0*__.adoc|.xml` 与 `pages-zh/0*__.adoc|.xml`，建立英文↔中文配对清单。
2. 逐章核对（第5章开始）：
   - `.adoc`：标题、概览、学习目标、章节与小节、工作流示例、说明与注意事项、练习逐段对照修正。
   - `.xml`：验证 DocBook 生成内容含义与结构与英文一致；修正可能的转义/符号错误与未解析指令标记。
3. 符号规范化批修：统一修复常见符号与转义（`+%`、`+|`、`[*]T`、`[:0]T`、`pass:[...]`、反引号/方括号转义）。
4. 交叉引用与链接：确认所有 `link:` 与 `xref` 指向正确章节与锚点；保持中文锚点标题与英文匹配。
5. 质量验证：
   - 自检渲染：检查 `.adoc` 中包含指令 `include::`、代码块语言、表格列头；
   - 站点预览（确认后执行）：本地预览中文站点逐页验证渲染、示例、表格与图。
6. 结果输出：按章节列出修复项与修改文件路径，给出差异摘要与潜在后续优化建议。

## 统一术语与风格
- 章节名示例：`Overview → 概览`、`Learning Goals → 学习目标`、`Notes & Caveats → 说明与注意事项`、`Workflow Patterns → 工作流模式`。
- 示例标签：`.Run → .运行`、`.Output → .输出`；保持命令/输出内容英文原样。
- 保持代码与类型名英文原文（如 `std.debug.print`、`@alignCast`）。

## 验证与交付
- 通过预览逐章人工验收：结构一致、无未解析指令、符号渲染正确、示例可读一致。
- 交付清单：
  - 章节修复列表与变更文件路径
  - 统一术语表与易错符号清单
